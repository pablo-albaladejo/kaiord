# ESLint Configuration

This project uses ESLint v9 with the flat config format to enforce code quality standards.

## Key Rules

### File Length Limits

- **max-lines**: 100 lines per file (excluding blank lines and comments)
  - Enforces the architectural principle of small, focused modules
  - Automatically disabled for test files (`*.test.ts`, `*.spec.ts`, `tests/**/*.ts`)
  - If a file exceeds this limit, split it into smaller, focused modules

### Function Length Limits

- **max-lines-per-function**: 40 lines per function (warning level)
  - Encourages Single Responsibility Principle
  - Automatically disabled for test files

### TypeScript Rules

- **@typescript-eslint/no-explicit-any**: Error on explicit `any` types
- **@typescript-eslint/no-unsafe-\***: Error on unsafe type operations
- **@typescript-eslint/consistent-type-definitions**: Enforce `type` over `interface`
- **@typescript-eslint/consistent-type-imports**: Enforce separate type imports

## Usage

```bash
# Check all files (ESLint + Prettier)
pnpm lint

# Check each package individually
pnpm lint:packages

# Auto-fix issues where possible
pnpm lint:fix

# Format code with Prettier
pnpm format

# Check formatting without making changes
pnpm format:check
```

## Ignored Patterns

The following directories are automatically ignored:

- `dist/` - Build output
- `coverage/` - Test coverage reports
- `node_modules/` - Dependencies
- `*.config.js` and `*.config.ts` - Configuration files

## Overriding Rules

Test files automatically have relaxed rules:

- No file length limits
- No function length limits

This allows comprehensive test coverage without artificial constraints.
