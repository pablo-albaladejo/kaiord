## Why

A guideline-vs-code audit (May 2026) surfaced eight drift points where the documented rules in `.claude/skills/guidelines/` are not mechanically enforced. The most damaging gap is that `architecture-hexagonal` already names a `check-architecture.js` pre-commit hook, but the script does not exist — layer boundaries are honored only by convention. Three other rules (mapper-vs-converter, conventional-commit type vocabulary, no-skip-tests) live in markdown alone, and silent drift (`.mapper.ts` files with tests, `.converter.ts` files without, `openspec:` used as a commit type, five unconditional `it.skip` calls) has already accumulated. This change makes every guideline rule that can be a script, lint, or commitlint be one — closing the gap between documentation and enforcement before more drift accrues.

## What Changes

- **NEW** `scripts/check-architecture.mjs` (+ `.test.mjs`) + `scripts/architecture.vocab.mjs` (single source-of-truth `CORE_ADAPTER_ALLOWLIST = ["analytics","logger"]` array): static-source AST guard fulfilling the `Domain Purity`, `Application Isolation`, `Layer Hierarchy`, `Port Contracts`, `Adapter Freedom`, and `Package Dependencies` scenarios already declared (or modernized) in the `hexagonal-arch` spec. Excludes `*.test.{ts,tsx}`, `*.spec.{ts,tsx}`, `*.stories.{ts,tsx}`, `dist/`, and `node_modules/` from layer evaluation so test files MAY import `vitest` and `@kaiord/core/test-utils` legitimately. The `<!-- arch-vocab -->` block in `architecture-hexagonal/SKILL.md` is byte-tested for drift against `architecture.vocab.mjs`. Wired into `pnpm test:scripts`, the husky `pre-commit` hook, and `pnpm lint`.
- **NEW** `scripts/check-package-deps.mjs` (+ `.test.mjs`): reads every `packages/*/package.json` and rejects any `@kaiord/*` workspace dep not on the allowlist for that package; allowlist mirrors the `Package Dependencies` table in the `hexagonal-arch` spec. Closes the documented-but-unenforced package-dependency table.
- **NEW** `commitlint.config.mjs` + `commitlint.vocab.mjs` (single source-of-truth module) + `.husky/commit-msg` + `scripts/check-commitlint-config.test.mjs`: enforces the documented type vocabulary (7 types: feat, fix, chore, test, docs, refactor, perf) and the audited scope allowlist (24 scopes; canonical order is the alphabetical-by-tier form spelled out in `commitlint.vocab.mjs` and reproduced in `git-strategy/SKILL.md`'s `<!-- commitlint-source-of-truth -->` block — the flat enumeration here is informative only; tasks 1.7.2 and 1.7.4 hold the canonical ordered form: core, fit, tcx, zwo, garmin, garmin-connect, ai, cli, mcp; spa-editor, garmin-bridge, train2go-bridge; analytics, landing, docs-site; openspec, ci, docs, scripts; deploy, release, deps, deps-dev, e2e). The scope list is parsed from the SKILL.md block and `assert.deepStrictEqual`'d against the `commitlint.vocab.mjs` arrays, so doc/config drift (insertion, deletion, OR reorder) hard-fails. Multi-scope subjects (`refactor(core,fit,tcx): ...`) are REJECTED per design D2 (single-scope is normative); the test asserts this rejection.
- **NEW** `scripts/check-mapper-no-tests.mjs` (+ `.test.mjs`): fails when any `**/*.mapper.test.{ts,tsx}` exists. Codifies the `testing-standards` rule that mappers MUST NOT have tests.
- **NEW** `scripts/check-converter-has-tests.mjs` (+ `.test.mjs`): fails when any `**/*.converter.{ts,tsx}` lacks a sibling `*.converter.test.{ts,tsx}`. Codifies the inverse `testing-standards` rule.
- **NEW** `scripts/check-no-unconditional-skip.mjs` (+ `.test.mjs`): rejects any `*.skip`, `*.only`, `*.todo` access on `it`/`test`/`describe` across all four dispatch shapes (member, computed-member, destructured, re-bound). Allows `*.skipIf(<expr>)` ONLY when `<expr>` contains at least one runtime-evaluated identifier reference, function call, or member-expression read — `skipIf(true)` and other literal-only arguments are rejected as functionally equivalent to unconditional skip.
- **NEW** `scripts/check-husky-no-bypass-hint.mjs` (+ `.test.mjs`): rejects only IMPERATIVE-VOICE bypass instructions (`echo "use --no-verify"`, `printf "HUSKY=0 ..."`) — defensive comments such as `# NEVER use --no-verify; CI re-runs all checks` are EXPLICITLY allowed. Distinguishes endorsement from prohibition via line-level pattern matching. `--no-gpg-sign` is OUT OF SCOPE for this rule (separate signing concern; deferred to a future `R-NoGpgBypass` rule if mandatory GPG signing is adopted).
- **NEW** `scripts/check-allowlists-empty.mjs` (+ `.test.mjs`): permanent invariant after archive — any non-empty `ALLOWLIST = new Set([...])` literal in any `scripts/check-*.mjs` fails CI (rule `R-AllowlistsEmpty`). Wired in `--mode=warn` during PR1 (because seed allowlists are intentionally non-empty during the cleanup PRs) and flipped to `--mode=error` during the final-validation block before archive. Codifies that no future PR may re-seed any drained allowlist without an explicit OpenSpec amendment to this change.
- **REORDER** `.husky/pre-commit`: move `pnpm test:scripts` BEFORE `pnpm test`, so an `it.only` or `it.skip("only the selected test")` is rejected before the test runner has a chance to mask the broader suite.

## Impact

**Affected packages**: `@kaiord/core` (types/ folder removal), `@kaiord/fit` (types/ folder addition), `@kaiord/garmin`, `@kaiord/tcx`, `@kaiord/zwo`, `@kaiord/workout-spa-editor` (mapper→converter renames + un-skip fixes), repo root (`scripts/`, `.husky/`, `commitlint.config.mjs`, `package.json` devDeps).

**Public API**: no runtime changes. Mapper→converter renames are internal to each adapter package; barrel exports keep the same exported symbol names. The factory exports of each affected adapter are unchanged.

**CI/CD**: `pnpm test:scripts` gains 8 new checks (architecture, package-deps, mapper-no-tests, converter-has-tests, no-unconditional-skip, husky-no-bypass-hint, commitlint-config-test, allowlists-empty). `pnpm lint` (root) gains the same 7 checks. `.husky/pre-commit` runs `pnpm test:scripts` BEFORE `pnpm test` (reordered) so `it.only` is caught before the test runner masks the broader suite. `.husky/commit-msg` is new and runs `pnpm exec commitlint --edit "$1"`. The existing GitHub Actions `ci.yml` workflow already runs `pnpm test:scripts`; this change verifies and (if absent) adds a `pnpm exec commitlint --from <merge-base> --to HEAD` step so a `--no-verify`-bypassed commit cannot escape detection at merge time.

**Breaking changes**: none for consumers of `@kaiord/core` or any adapter package. New repo-internal guards may reject in-flight branches that do not yet conform — these branches MUST be rebased and fixed (the rules being enforced are already documented as MUST in the existing guidelines).

**Dependencies**: adds dev deps `@commitlint/cli`, `@commitlint/config-conventional` at the repo root.

**Changesets**: only PRs that change the public API of a published `@kaiord/*` package generate a changeset. PR1 (root tooling), PR2 (FIT type relocation, internal), and PR4 (test/docs only) MUST NOT add a changeset. PR3 (mapper→converter renames) MUST NOT add a changeset because the rename is internal — exported symbol names and the package barrel API are unchanged. This matches the `git-strategy/SKILL.md` rule that a changeset is only required when a published package's behavior changes.

**Tests**: every new script ships with a `node:test` co-located test (`*.test.mjs`) per the existing `scripts/` convention; the husky `pre-commit` hook exercises them. Coverage thresholds (80% core, 70% frontend) MUST hold after the mapper→converter renames; verified in the final validation block.

**Specs touched**: `hexagonal-arch` (delta with `## MODIFIED` and `## ADDED` requirements), `spa-quality-gates` (delta with `## ADDED` requirements only — the existing toast/PII/Zustand requirements are untouched). No new capability specs.
