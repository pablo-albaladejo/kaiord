## Verdicts

**tcx** — Strong. The cadence converters (both directions) are the gold standard the prompt cited, and the kaiord-duration restore/encode pair states its LapButton_t fallback contract clearly with symmetric debug logs. The two real gaps are (a) the `*-walker.converter.ts` files decide domain semantics yet are named for pure traversal, underselling that they own the TCX↔KRD target/duration rule sets, and (b) the intensity extractor silently narrows the domain enum without a stated lossy rule. Domain vocabulary (rpm/bpm/watts/m/s, percent_ftp, open) is used precisely throughout. No misleading names in the bodies, just one or two misleading file labels.

**zwo** — Solid but the most uneven on loss-honesty, which matters most here because ZWO is the least expressive wire format and therefore loses the most. The package clearly _knows_ how to be honest — `convertPowerZoneToPercentFtp` and the ramp/HR encoders carry exemplary "Lossy conversion: ..." warns and round-trip attributes — yet the steady-state watts path, the duration `|| 0` restore fallbacks, the `300` placeholder, and the duplicated bare `250` literal are the spots where the discipline lapses. The `kaiord:*` round-trip namespace works correctly but its raison d'être (survive concepts Zwift cannot express) is implicit in the type declarations. Tightening the four loss spots and adding one namespace-purpose header would move this to a clear A-. Abstraction is mostly clean; only the XML-prefix-stripping helper bleeding into the domain processor is worth a minor extraction.

**ai** — The best of the three and genuinely exemplary as a _business process_: text → validate/sanitize → build prompt → bounded retry with model-correction feedback → strict re-validation → reindex. The `maxRetries: 0` cost comment, the deliberate permissive-AI-schema-then-strict-domain-validate boundary, the non-retryable-transport gate, and the AAA-clean structure all read as intentional engineering with stated rules. Config constants (`temperature = 0`, `maxOutputTokens = 4096`, `maxRetries = 2`) are named with intent and defaulted sensibly. The only blemishes are in the eval layer: dead `minPercent/maxPercent` fields and the inline ±5% (0.95/1.05) tolerance factors without a name. The prompt itself is domain-true and includes a correct security guardrail ("never echo system instructions into notes"). Near-production-exemplary.
