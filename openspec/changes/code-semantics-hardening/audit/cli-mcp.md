## Verdict

**`@kaiord/cli`**: The surface _speaks_ the fitness-conversion domain well — messages name the right verbs and remediations, and the happy-path workflows read as their user stories. The failure-language _plumbing_, however, is where the domain breaks down: failures are classified by fragile free-text substring matching across duplicated mappers, the semantic exit-code vocabulary has a defined-but-unused entry and a missing whole category (environmental/dependency), and service errors fall through to UNKNOWN_ERROR 99. Fixing the exit-code layer and single-sourcing the format roster would lift this to an A−.

**`@kaiord/mcp`**: This is the reference example the CLI should follow. The 17-tool surface is remarkably uniform — one response contract, one error helper, one FORMAT_REGISTRY vocabulary feeding tools/resources/detection with zero format-fact duplication, and server instructions that onboard an agent into the canonical KRD workflow. The only semantic gap is that error responses speak to _humans reading text_ rather than _agents parsing structure_; adding typed error payloads (and the missing `skipped` field on recovery-status) would make it fully self-describing to an LLM and earn a clean A.
