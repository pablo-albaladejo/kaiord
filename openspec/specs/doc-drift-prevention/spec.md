# Doc Drift Prevention

## Purpose

Safeguards — type-checked code examples, TypeDoc generation, link checking, spellcheck, and a doc-sync hook — that keep the documentation faithful to the code it describes.

## Requirements

### Requirement: Twoslash type-checked code examples

All TypeScript code examples in documentation SHALL be compiled against real `@kaiord/*` package types at build time using `@shikijs/vitepress-twoslash`. The docs build SHALL fail if any code example contains type errors.

#### Scenario: Code example matches current API

- **WHEN** the docs build runs with valid code examples
- **THEN** the build SHALL succeed and code blocks SHALL render with type information tooltips

#### Scenario: Code example drifts from API

- **WHEN** a package export signature changes and a doc code example uses the old signature
- **THEN** the docs build SHALL fail with a TypeScript compilation error identifying the broken example

### Requirement: TypeDoc auto-generated API reference

API reference documentation SHALL be auto-generated by TypeDoc from TSDoc comments at build time (prebuild step). Generated Markdown files SHALL be gitignored to avoid noisy PR diffs.

#### Scenario: API reference regeneration

- **WHEN** the docs prebuild script runs
- **THEN** TypeDoc SHALL regenerate API reference Markdown files from current source TSDoc comments

#### Scenario: Generated files not committed

- **WHEN** a developer runs the docs build locally
- **THEN** the `api/` directory SHALL be in `.gitignore` and not appear in `git status`

### Requirement: CI link checker

The CI pipeline SHALL check all internal links in documentation pages. Broken links SHALL fail the build.

#### Scenario: Broken link detected

- **WHEN** a documentation page references a non-existent internal path
- **THEN** the CI link check SHALL fail and report the broken link

### Requirement: CI spellcheck

The CI pipeline SHALL run spellcheck (cspell) on documentation content. Unrecognized words SHALL fail unless added to a project dictionary.

#### Scenario: Typo detected

- **WHEN** documentation contains a misspelled word not in the project dictionary
- **THEN** the CI spellcheck SHALL fail and report the typo

### Requirement: Claude Code doc-sync hook

A Claude Code pre-commit hook SHALL check `git diff --cached --name-only` — if source files in `packages/*/src/` are staged but no `packages/docs/` files are staged, it SHALL display a reminder. If docs files are also staged, it SHALL pass silently.

#### Scenario: Source changed without docs

- **WHEN** a developer commits changes to `packages/core/src/` without staging any `packages/docs/` files
- **THEN** the pre-commit hook SHALL display a reminder to check if docs need updating

#### Scenario: Source and docs changed together

- **WHEN** a developer commits changes to both `packages/core/src/` and `packages/docs/`
- **THEN** the pre-commit hook SHALL pass silently
