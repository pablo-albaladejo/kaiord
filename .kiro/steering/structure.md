## Project Structure

```
kaiord/
├─ packages/
│  ├─ core/          # @kaiord/core - Core library
│  │  ├─ src/        # Schema definitions and converters
│  │  └─ dist/       # Build output (gitignored)
│  └─ cli/           # @kaiord/cli - Command-line interface
│     ├─ src/        # CLI implementation
│     └─ dist/       # Build output (gitignored)
├─ .kiro/            # Kiro configuration
│  ├─ steering/      # AI assistant guidance docs
│  ├─ specs/         # Feature specifications
│  └─ hooks/         # Automated validation hooks
├─ LICENSE           # MIT License
├─ NOTICE            # Copyright notice
├─ README.md         # Project documentation
└─ pnpm-workspace.yaml  # Monorepo configuration
```

## Conventions

- **Monorepo**: pnpm workspaces with packages under `packages/`
- **Build artifacts**: All `dist/` and `node_modules/` directories are gitignored
- **Kiro integration**: The `.kiro/` directory is version-controlled and contains steering docs, specs, and hooks
- **Licensing**: MIT licensed with third-party licenses tracked in THIRD-PARTY-LICENSES.md

## Package Organization

Each package follows a standard structure:

- `src/` - Source TypeScript files
- `dist/` - Compiled output (generated by tsup)
- Package-specific configuration files (tsconfig.json, package.json)
