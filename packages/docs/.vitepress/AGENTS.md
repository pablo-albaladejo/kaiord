# .vitepress

## Purpose

VitePress configuration and theme customization. Houses the site config, routing, SEO metadata transforms, and Vue components for the documentation site.

## Key Files

- `config.ts` — Central VitePress configuration: base path, site title, description, nav/sidebar structure, SEO head transforms (JSON-LD breadcrumbs, OpenGraph), appearance, themeConfig, Markdown extensions (Shikijs twoslash), Vite build settings
- `head-config.mjs` — Factory function to build static `<head>` metadata (charsets, fonts, prefetch)
- `brand-tokens.mjs` — CSS custom property definitions for design tokens (colors, spacing, typography)
- `public/` — VitePress-local static assets (separate from `../public/`)
- `theme/` — Custom Vue theme components and styling
  - `NotFound.vue` — Custom 404 page layout
  - `custom.css` — Global CSS and overrides (design system colors, layout, responsive)
  - `index.ts` — Theme setup entry point (initializes VitePress theme)
- `dist/` — Build output (gitignored; generated by `vitepress build`)

## Subdirectories

- **`theme/`** — Custom Vue components, CSS, and theme initialization
- **`public/`** — Site-local assets (separate scope from root `public/`)
- **`dist/`** — Build output (not checked in; generated at build time)

## For AI Agents

### Working In This Directory

1. **config.ts** — Source of truth for nav structure, sidebar items, SEO transforms. Changes here affect all pages. Modify only for structural changes (adding new sections, SEO improvements, plugin integration).
2. **theme/** — Custom styling and components. Use `custom.css` for global design system; extend `NotFound.vue` for 404 behavior.
3. **head-config.mjs + brand-tokens.mjs** — Read-only during normal development. Modify only if adding new CSS custom properties or static head tags.

### Testing Requirements

- **Tests in `../scripts/`**:
  - `head-config.test.mjs` — Validates `head-config.mjs` output (presence of required tags, charsets, etc.)
  - `brand-tokens.test.mjs` — Validates `brand-tokens.mjs` exports (all tokens are CSS-valid)
- **VitePress build**: `pnpm --filter @kaiord/docs build` must complete without warnings or errors.
- **Link validation**: After changes to `config.ts` sidebar/nav, run build and verify no broken links.

### Common Patterns

- **SEO transforms** (in config.ts): The `transformHead` function inspects page frontmatter and injects JSON-LD breadcrumbs + OpenGraph tags. This is critical for docs-site discoverability.
- **Sidebar nesting**: Items in `themeConfig.sidebar` can use `collapsed: false` for expanded sections or `items: [...]` for sub-pages.
- **CSS custom properties**: Define in `brand-tokens.mjs`, reference in `custom.css` and Vue components as `var(--vp-c-brand-1)`, etc.
- **Theme imports**: `index.ts` initializes the theme; it imports VitePress default theme and applies custom CSS/components.

## Dependencies

### Internal

- `../brand-tokens.mjs` — Used by brand-tokens.test.mjs; could be imported by theme CSS

## Notes for Agents

1. **config.ts controls everything visible**: Sidebar structure, nav links, search config, footer, social links. Every docs update likely requires a sidebar entry here.
2. **dist/ is transient**: Never commit or hand-edit. It is regenerated on every `pnpm build`.
3. **SEO is baked into config**: JSON-LD breadcrumbs and OpenGraph transforms are in the `transformHead` hook. Ensure all pages have frontmatter `title` and `description`.
4. **Twoslash integration**: Code examples with ` ```ts twoslash ``` ` are processed by the `transformerTwoslash` plugin to show inline type hints. This works because config.ts enables it.
5. **Theme customization is limited**: VitePress theme is mostly CSS-driven. For major structural changes (e.g., adding new header components), consider forking the default theme or using VitePress plugins.
